# Security Policy

This document explains how to report security vulnerabilities in this repository.

---

## Reporting a Vulnerability

If you discover a security issue, **please do NOT open a public issue**.  

Instead, report it privately by emailing:

**[iamlucid27@gmail.com]**

When you report a vulnerability, you can expect:

- Acknowledgment within **48 hours**  
- Regular updates on the progress of the fix  
- Credit in the repository when the issue is resolved (optional)

Please include as much detail as possible, such as:

- Steps to reproduce the issue  
- Impact and severity  
- Environment details (OS, software version, etc.)

---

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Fully supported |
| older   | ❌ No longer supported |

> Note: Security fixes will only be applied to supported versions. Please ensure you are using the latest version.

---

## Security Best Practices for Users

To reduce risk, we recommend:

- Keep your software up-to-date  
- Avoid exposing sensitive data publicly  
- Follow recommended configuration and usage guidelines

Thank you for helping us keep this project secure! 🔒
